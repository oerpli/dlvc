In Part 2 you’ve seen that the kNN classifier performs poorly on image data. This is mainly because it is unable to find discriminative features on its own. The first task is to confirm experimentally that this is the case by validating the classifier on features extracted from images instead of these images themselves.

For this purpose, I’ve extracted the popular HOG features from all images in the TinyCifar10Dataset. You can download these features here (train, validation, test). The files are in HDF5 format, a database format that is supported by all languages (Python, Matlab, Java, Lua). Each file contains two databases, features and labels. The former is a matrix of size n*144, with n being the number of images in the original dataset. The ith row in this matrix corresponds to the extracted HOG features of the ith sample in the original TinyCifar10Dataset. labels is a n-vector of labels for every sample.
